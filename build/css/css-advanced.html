Responsive CSS:

Using media queries we can achieve any layout we want on the right device. In the previous tutorial, we learned what media queries are. Now, we'll dig in and put them to use.

To make media queries work, we need to add a meta tag to our HTML head, like the following:

  • CSS
<meta name="viewport" content="width=device-width, initial-scale=1">

The above meta tag simply tells the browser to resize our browser width according to the device width, whether it is a mobile phone screen or large monitor.

In the following JSFiddle snippet, we have 2 columns. On desktop view they are side by side but on mobile they stack one below the other. We do this by using a media query that changes the width of the content from 50% to 100% on mobile. We also add a small margin-bottom to separate the column and improve readability.

Link: JSFiddle - 2 column example.

This example demonstrates the basic capability of the media query. There are, however, certain limitations. It is not possible to change HTML layout with just CSS. So, for example, if a menu was above the content block in the HTML, it won’t be possible to show it below the content block without some visual tricks involving hard-coding heights and using absolute positioning. But in most cases, you can avoid this simply by changing the location of the element in the HTML.

In general, pages should be designed to look good for mobile and desktop, as well as keep a visual ordering of elements on the page consistent in both environments. Responsive design takes all of these consideration into account and seeks to provide a consistent experience across all platforms.

CSS3 Browser Support:

While most modern browsers support CSS3, outdated browsers like Internet Explorer 8 and 9 do not. Fortunately, newer versions of IE (IE10 and 11) do support CSS3. When developing CSS for the web, it's important to understand what browsers your user demographic is using. Older age groups and large businesses are more likely to use Internet Explorer and in some cases, outdated versions of IE. If you find yourself needing to support outdated browsers, it's crucial to understand which CSS properties are supported.

There are many ways to know if a specific property is supported. One of the most widely used tools is caniuse.com. Here, you'll find an up-to-date, thorough resource.

CSS3 Transform:

The CSS3 transform property allows us to change an element’s appearance and position without actually moving it from its original position within the page. In the below example we will talk about different ways we can use the transform property. Below, you'll see an example of each these properties in action.

Scale

Using transform: scale(.8); we can scale our element down to 80% of its original size. We also use this property to make shapes large. For example 1.5 would make our square 50% larger.

Rotate

By using transform: rotate(45deg);, we can rotate our square 45 degrees to create a diamond shape.

Multiple properties on the same element

The translate property moves an element from its position depending on the y-axis and x-axis values, transform: translate(x, y).

We can combine as many transform properties as we want. In this example, we'll use transform: rotate(45deg); to rotate our square and give us a diamond shape. Then, we'll use transform: translate(0, 50px) to move our diamond shape 50px to the bottom.

Alternatively, we can use percentage values on our transforms. With percentage values, the transform will be based on the element's height and width. For example, if we have a 100px by 100px square and we use translateX(100%), our element will go to the right 100px because our square's width is 100px.

Perspective:

When the perspective property is applied to an element, its children will be treated as 3D objects. This gives us the ability to perform various 3D transformations.

CSS3 Animations:

Simply put, CSS animations are merely transforms with keyframes. A typical keyframe animation looks like this:

  • CSS
@keyframes animation-expand {
  0% {
    transform: scale(1.5);
  }
  50% {
    transform: scale(1);
  }
  100% {
    transform: scale(1.5);
  }
}

.animation-expand {
  animation: 1s animation-expand linear infinite;
}

With the @keyframes rule, we can split up an animation by using percentage values. In the example above, we've created three keyframes at 0%, 50%, and 100%. Within each keyframe, we use the transform property we just learned about to manipulate the shape. This rule allows for as much complexity as you want - meaning we can combine several different transforms into one animation.

Beneath @keyframes rule, we've defined the .animation-expand class. This class uses the animation property to set the keyframes in motion. Read more about the animations property here.

Getting started:

The layout we are building is very simple. Essentially, we are building a container div, three columns of content inside of it, and a "sticky" topbar that will stick to the top of the screen on mobile devices. This layout, though simple, becomes more complex when it must work on smaller screens as well.

Our Markup:

We'll be using dummy text as our content for now. First, we need to apply our classes and divs to make the layout process easier. We are using a 12 column grid system to define the layout. Since we only want three columns, each column will have 4 smaller columns within it (making a total of 12 columns). So we'll use .col-4 and apply a float: left property to our columns along with a clearfix. Clearfix is a common hack that forces an element to self-clear its children. This allows us to create the columns layout we're looking for.

On mobile, we use a media query to change two aspects of our layout:

Our columns:

We need to apply a 100% width to all of our columns once they get into our mobile size. This will result in our columns taking up 100% of the screen width whereas normally they take up 33.333% of our container. Again, this is a technique to make our content more responsive and legible on a smaller screen.

Our topbar:

First we'll give our container a padding-top based on our topbar's height. Next, we'll position our topbar fixed to the top and left positions, while giving it a 100% width. By positioning our topbar this way, we are taking it out of the view. Now, whenever the user scrolls the topbar will always stay on the top part of the page. By applying an opacity to the element, users can see the content below it once they start scrolling.

Helping Classes:

Lastly, we have to define two helping classes. .show-mobile this shows an element on mobile and hides it from desktop view, by default it's set to display: none; and once we reach our mobile view, we set it to display: block;. Next, we have .hide-mobile which simply hides any element at our mobile width.
