Terms and Defintions:

Let's get started by learning some basic terminology.

Cascading Style Sheets (CSS) is a style sheet language used for describing the look and formatting of a document written in a markup language. While most often used to change the style of web pages and user interfaces written in HTML and XHTML, the language can be applied to any kind of XML document, including plain XML, SVG and XUL. Along with HTML and JavaScript, CSS is a cornerstone technology used by most websites to create visually engaging web pages, user interfaces for web applications, and user interfaces for many mobile applications.

Link: Wikipedia - CSS


CSS3 is the latest evolution of the Cascading Style Sheets language and aims to extend CSS2.1. The term “CSS3” is not just a reference to the new features in CSS, but the third level in the progress of the CSS specification. CSS3 contains just about everything that’s included in CSS2.1 (the previous version of the specification). It also adds new features to help developers solve a number of problems without the need for non-semantic markup, complex scripting, or extra images.

Link: w3 - Current Work


The Document Object Model (DOM) is a cross-platform and language-independent convention for representing and interacting with objects in HTML, XHTML and XML documents. The nodes of every document are organized in a tree structure, called the DOM tree.

Objects in the DOM tree may be addressed and manipulated by using methods on the objects using application programming interface (API).

Link: CSS Tricks - Source


Common CSS3 properties:

  1. border-radius
    • Allows creation of rounded corners without use of images.
  2. transform
    • Various dimensional transformations (rotate, stretch, etc).
  3. transition
    • Ability to change CSS values smoothly, used in animations.
  4. animation
    • Ability to animate any element with help of transitions.
  5. overflow, overflow-x, overflow-y and overflow-style
    • Control how overflowing content is displayed.
  6. box-shadow
    • Shadows.
  7. opacity
    • Transparency for elements.

The Role of the Browser:

When working with CSS, it is important to be aware of the user's browsing environment. The size of the screen and the kind of browser are the most important traits of the user's browsing environment. As of May 2015, the majority of people use Google Chrome (65%) and Firefox (21.5%). Each browser renders a web page slightly differently and it is important to understand these differences in order to create a consistent experience across all major browsers.

Note: W3 - Source


CSS3 Rendering engines:

Each browser has its own rendering engine. For example, Chrome and Safari use Webkit, Firefox uses Gecko, older versions of Internet Explorer use the Trident engine, and the new IE rendering engine has code name Spartan.

Engines vary by CSS compatibility levels, performance, and features supported. It is highly recommended to test websites in all major browsers. Tools like BrowserStack and Sauce Labs help you conduct cross-browser testing quickly and efficiently. Using prefixes and media queries (covered below), you can create CSS that addresses each rendering engine's idiosyncracies.

Link: Wikipedia - Source


Browser Prefixes:

  1. Chrome, Safari and any browser using webkit engine : -webkit-
  2. Firefox : -moz-
  3. Internet Explorer : -ms-
  4. Opera : -o-
For example:

At first, the border-radius property was not fully articulated in CSS3, so Chrome used -webkit-border-radius and Firefox used -moz-border-radius. After the final acceptance of border-radius by the W3C committee, all CSS3-compatible browsers support the standard and prefixed versions for maximum compatibility.

The same thing applies to all properties that are not yet finalized by the W3C committee.


CSS3 Media Queries:

A Media Query consists of a media type and at least one expression that limits the style sheet's scope by using features such as width, height, and color. Media queries were added in CSS3 and let the presentation of content be tailored to a specific range of output devices without having to change the content itself. This technique is crucial to creating CSS that adapts to different screen sizes.

Link: MDN - Source.

Link: CSSTricks - Source.

Code Example:
  • CSS
@media (min-width: 700px) and (max-width: 900px) {
  CSS rules here...
}

Note: Any rules applied in the previous Media Query will only work if the screen width is between 700px to 900px.

For example, if we apply a background-color: red; to our body tags inside the previous Media Query we made, the body will have a red background only when the screen width is between 700-900px. If our viewport (screen size) increases or decreases outside of our 700-900px range, our body's background will go back to its default value (white).

  • CSS
@media (min-width: 700px) and (max-width: 900px) {
  body {
    background-color: red;
  }
}

Performance:

On a high level, CSS is all about applying styling rules to DOM nodes using selectors. Within each selector, there are properties (discussed earlier). The more rules there are, the more computationally expensive that page is to render. For this reason, performance is an important factor to consider when creating CSS.

Tips for improving CSS performance:
  1. Eliminating dead code helps the most.
  2. Be very careful with “heavy” selectors. For example substring matching [class*=foo] is significantly slower than direct string matching, like .foo.
  3. Try keeping selectors short. Deeply nested selectors hurt performance, as browser will have to traverse tree every time when DOM node is created. If matching is happening on a, lets say, class - it is much simpler test to do.
  4. Use debugging tools to see how long things take.

Note: You can use Chrome dev tools to see how long Chrome spent parsing the page, applying CSS rules, etc. Also, Chrome's continuous painting mode will show how the page is rendered in slow motion and the rendering sequence of different elements.

Link: Ben Frain Source.

Link: Shay Howe Source.

CSS3 Hardware acceleration:

With additional computational complexity that was introduced with CSS3 (animations, 3D transforms, dynamic shadows, etc) it was clear that it will be very hard to achieve smooth performance without help of the GPU.

Note: GPU stands for Graphical Processing Unit - a special device that works together with CPU (Central Processing Unit) to speed up various graphics related tasks.

In general, translate and translate3d rules are hardware-accelerated in modern browsers. The browser will use the GPU as much as possible to create a smooth user experience without putting a burden on the CPU.
