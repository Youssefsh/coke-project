SVG Beginner

SVG is short for Scalable Vector Graphics. It is a graphic format in which shapes are specified in XML and rendered by an SVG viewer. Today, most web browsers can display SVG just like they can display PNG, GIF, and JPG. You can use SVG on the web pretty easily, but there is plenty you should know.

In this tutorial, you'll get an understanding of why SVG is useful, what browsers are compatibile with it, how to use it on your website, how to create SVG, and how to export it from Adobe Illustrator. Anything graphic made with vector based tools like Adobe Illustrator, Sketch, and Inkscape can be easily be converted to SVG. Once you have a grasp of how SVG works generally, we'll move on to SVG Intermediate where we'll focus on animating SVG with CSS3, Javascript, and SMIL. Once you feel ready, we'll progress to the SVG Final Project where you'll learn how to create a Coke contour bottle loader!

But let's not get ahead of ourselves. First we must understand what SVG is and why it's useful. Let's get started!

Why SVG?:

In the early days of the web (mid 90's), it was clear that the vector format would be useful for the web. Several large companies like Adobe, IBM, Netscape, Microsoft, HP, and others published competing specifications for how web vector graphics should work. In 1998, the W3C examined all the submissions and developed SVG WG, thus establishing a universal standard. SVG became increasingly popular and when Adobe's Flash was denounced by Apple in 2008, SVG began its rapid rise in popularity. With the increase in variety of screen sizes and resolutions, SVG's adaptability and built-in browser support cemented its position as the gold standard in web vector graphics.

Link: The Secret History of SVG.

But why is SVG so good? What makes it useful?

  • Editable: Each and every element and attribute in SVG's can be edited and animated by means of your favorite editor.
  • Scalable: SVG's can be zoomed, enlarged or scaled down without limit whereby the quality will always remain (with the exception of extremely small).
  • Retina: Big or small, with SVG you will never see pixels. This makes SVG perfect for retina screens.
  • Loading time: When SVG's aren’t too complex the file size will be relatively small and easy to compress.
  • SEO: Search engines index and cache SVG files . Therefore search engines can be used to search for SVG files.
  • Perfect display: You can make SVG's interactive and make use of filters, gradients and other formats to ensure the image is always properly displayed.
  • Fallback: When SVG's aren’t supported by a browser or other device, there are various ways to create a fallback.
  • Recommended by the W3C: The W3C recommends the use of SVG and SVG easily integrate with other W3C standards like DOM and XLS.

Browser Compatability:

All modern browsers support the use of SVG's. Only the older Internet Explorer versions (IE8 or lower) and the older Android (2.3 and lower) do not support SVG. However, some technologies closely associated with SVG are not as widely supported. SMIL Animation, is not supported on any IE old or new. Use resources like caniuse.com to determine whether your code is cross-browser compatible.

Link: caniuse.com.

If you'd like to use SVG, but also need to support these browsers that don't support using SVG in this way, you have some options. You can use a tools like Modernizr and SVGeezy to swap out the src of an image or use David Bushell's simple Javascript alternative.

Link: David Bushell's JS.

Embedding SVG:

SVG's are ideal when a website is filled with images, background-images, logos, icons, patterns and basic shapes. SVG's are the perfect replacement for currently used jpg, png or gif images. When you are planning to use SVG files on your website, you should always question whether SVG file editing afterwards will still be required (with CSS or JavaScript) or if you would just want the SVG files to be displayed on your website without future editing requirements.

Embedding SVG is a simple process, these are three widely used methods to view SVG on a browser.

  • Inline SVG, by copying SVG code and pasting it directly into an HTML document.
  • Linking it in an
    img
    tag.
  • Applying it through CSS
    background
    property.
