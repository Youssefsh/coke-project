Intermediate SVG

Now that you have grasp of the SVG basics, it's time to move on to the fun part: animation. Because SVG is resolution independent, using SVG is a performant, cross-platform method to animate on the web. There are three different ways to animate SVG - each with its own advantages and disadvantages. Once you feel comfortable with animation, we'll move on to the Final Project where we'll learn how to build an animated Coke contour bottle loader.

In this tutorial, we'll learn about animating with CSS3, Javascript, and SMIL. In the mini tutorial, we'll dive in and create some simple animations. Let's start!

CSS3 Animation:

If you're using inline SVG, you can target the SVG (or shapes within) with CSS and animate using CSS animations. This method of animating SVG is the simplest and most approachable. Animating SVG with CSS is like animating any other element with CSS—it can be done with transitions, transforms, and keyframe animations. Once you’re familiar with the markup of an SVG, the rest is fairly straight forward.

However, the caveat with CSS transforms is cross-browser compatibility. Below is a list of discrepancies between browsers for CSS transform:

  • IE and Opera don't honor CSS transforms at all on SVG elements. Instead, you must assign the value to the transform attribute.
  • Firefox doesn't honor %-based origins.
  • Zooming in Safari breaks the sync between %-based and px-based origins.
  • In all browsers, px-based origins are measured differently for SVG elements than other DOM elements.

GSAP by GreenSock. Although every major browser offers excellent support for displaying SVG graphics, animating them is not always easy or consistent. Each browser has its own quirks and implementations of the SVG spec, causing quite a few challenges for animators. GSAP smooths out the rough spots and harmonizes behavior across browsers for you.

Velocity.js has gained popularity for extensive cross-browser SVG support and being lightweight. Velocity.js automatically detects when it's being used to animate an SVG element and seamlessly applies SVG-specific properties without you having to modify your code in any way.

The most popular SVG Javascript animation library is Snap.svg. Snap is the successor to Dmitry Baranovskiy's Raphaël and gets credit for support the most amount of animation capabilities. Snap was built to more easily use the latest that SVG can offer; features like masking, patterns, gradients, groups, animations and more.

SMIL Animation:

SMIL, short for Synchronized Multimedia Integration Language, allows you to animate SVG elements using SMIL specifications. The specifications resemble CSS transform but are supported by far more browsers. However, SMIL animations are not supported by Internet Explorer

Note: By using a SMIL polyfill we can add a fallback for Internet Explorer, example: Fakesmile.

SMIL Animation Elements:
  • <animate>, allows you to animate specific attributes and properties.
  • <set>, Short hand for <animate>.
  • <animateMotion>, Moves element along motion path.
  • <animateColor>, Modifies color values, - Deprecated.
  • <animateTransform>, Allows you to animate SVG transform attribute.
  • <mpath>, used in conjunction with the <animateMotion> element to reference a motion path that is to be used as a path for the motion.
  • keypoints, rotate ( attributes ), used as an attribute for <animateMotion>.

Mini Tutorial:

In this mini tutorial, we'll learn how to create some simple line and shape SVG animations. First we'll create the animations using CSS3 and the replicate them in SMIL. In doing so, you'll notice how each method is able to manipulate SVG shapes.

CSS animation of SVG:

Let's animate an SVG circle, square, and a line.

Let's start by creating keyframes animations for each one. We'll want to make all of our simple animations continuous. To achieve this, all we need to do is to add the same animation at the starting point of the keyframes 0% and the ending point of our keyframes 100%. If our animation ends and starts at the same transform, we'll have created a looped animation.

Note: by using the transform-origin property and setting it to 50% 50%, we are telling our element animate on its center axis rather than the default location in the upper left.

Circle:

Let's animate a circle by using transform: scale(value) translateY(value);. The scale() property simply scales our element based on its current size. By increasing and reducing our scale we are creating an a pulsating effect.

Square:

For our square, we'll rotate it using transform: rotate(45deg);

Line:

Tp animate the line, we'll use a couple different SVG properties in CSS, stroke-dasharray: value; and stroke-dashoffset: value;. Let's take a closer look:

1 - stroke-dasharray :

The idea here is to make the element's stroke-dasharray bigger than the element's length. By increasing the size of the dashes to be longer than our element's length, we are in effect hiding the line.

2 - stroke-dashoffset :

After applying our stroke-dasharray to be more than the line's length, let's use the stroke-dashoffset to offset our stroke. Let's also make it fill and unfill the line by adding stroke-dashoffset to some animation keyframes. The end result is a line drawing effect.
